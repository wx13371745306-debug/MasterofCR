## Master of CR - 多人合作经营策略模拟游戏（Demo）

这是一个专门用于游戏策划岗位求职的个人演示项目。

本项目旨在探索多人合作游戏在“动作交互”之外的“策略经营”深度，通过解耦的模块化架构实现了高度可扩展的玩法系统。

### 开发环境 (Environment)

Unity版本: 6000.3.10f1 (Unity 6 国际版)
网络框架: Mirror
开发模式: 独立策划并完成全栈程序开发（Client/Server）

### 注意事项:

若您在克隆项目后无法顺利打开，请务必检查 Unity 版本。由于 Unity 6 的底层差异，使用较低版本或中国版 Unity 可能导致项目无法加载或脚本丢失。

### 项目核心设计

这是一款融合了资源管理与动作交互的多人合作经营游戏。在项目中，我主导了核心玩法循环的设计与程序实现，将传统的“做饭闯关”拓展为具备深度策略的“餐厅经营”系统。有别于《胡闹厨房》等传统的关卡制时间管理游戏，我赋予了玩家更高的自由度与决策自由，将“做饭闯关”改为为真实的“合伙开店”。

本游戏主打多人联机，其核心的玩法是玩家与玩家之间相互合作处理订单的时间管理游戏。与传统的烹饪游戏（例如：《胡闹厨房》）不同的是，本游戏融合了更多特殊的系统，使其从一个合作闯关类游戏转变成了一个**合作经营类**的游戏。

本游戏中将选择“菜单”的权利交还给了玩家，设计了动态的菜单配置模块，底层支持多种菜谱的排列组合，不同菜谱间的“羁绊”数值增益不同，有的为单纯的增加数值，有的则是增加了游戏机制。这为玩家提供了**丰富的流派构筑（Build）体验**。

游戏还要求玩家去进行“库存管理”。本游戏摒弃了简单的无限食材生成，设计并实现了包含“**成本控制-动态消耗-保质期倒计时**”的复杂状态机系统，要求玩家在资金与时间压力下进行决策和管理。该系统提升了玩家的自由度，但也提高了游戏的难度与复杂性。

本游戏以多人联机为主，鼓励玩家多人游玩。为此设计了独特的“饰品”系统。该系统放弃传统的硬编码的职业类，开发了灵活的饰品组件系统。支持玩家自由插拔最多 2 个饰品装备“饰品”（如食材处理速度、冲刺特殊能力）。装备不同的“饰品”的玩家会在某些方面占有优势，通过该机制实现了**引导玩家自然形成差异化的团队分工。**

传统的做饭游戏都砍掉了“上菜”阶段，只需要将菜端到上菜口即可。但在本游戏中上菜则是一个玩家需要做的确切操作，必须由玩家本人去端到餐桌上服务顾客。这更加强调了合作的关系，除了做饭之外还要有人去“照顾”顾客。
